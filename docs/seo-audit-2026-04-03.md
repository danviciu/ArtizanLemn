# Audit Trafic & SEO - 3 Aprilie 2026

## Context
- Site: `https://artizanlemn.ro`
- Problema raportata: trafic organic foarte mic
- Obiectiv: crestere agresiva a indexarii utile si a vizibilitatii pe intentie comerciala

## Concluzie Audit
Traficul redus nu vine dintr-o eroare tehnica singulara, ci din combinatia:
- semnal semantic slab pentru Google (lipseau `JSON-LD` pentru `Organization`, `CollectionPage`, `Product`, `Breadcrumb`)
- arhitectura SEO limitata (filtrare pe query string pentru categorii, fara pagini indexabile dedicate per categorie)
- sitemap prea simplu (fara imagini, fara categorii dedicate, fara semnal de prioritate agresiva pe paginile care convertesc)
- lipsa motorului de continut editorial (nu exista cluster informational public pentru cautari top/mid funnel)
- lipsa semnalelor off-page (autoritate domeniu, citari locale, link-uri relevante)

## Date Tehnice Observate
- Lighthouse existent:
  - Home: `Performance 71`, `SEO 100`, `Accessibility 100`, `Best Practices 100`
  - Product: `Performance 88`, `SEO 100`, `Accessibility 100`, `Best Practices 100`
- Interpretare:
  - scorul SEO Lighthouse este bun, dar verifica doar un subset tehnic
  - lipsesc semnale de relevanta pe intentie + autoritate externa + acoperire de continut

## Implementari Aplicate in Cod (acest update)
- `JSON-LD` global in layout:
  - `Organization`
  - `WebSite`
- `JSON-LD` pe pagini comerciale:
  - `Product` + `BreadcrumbList` pe pagina de produs
  - `CollectionPage` + `BreadcrumbList` pe `/produse`
  - `CollectionPage` + `BreadcrumbList` pe `/categorii` si pe fiecare categorie
- Arhitectura SEO pe categorii:
  - pagini noi indexabile: `/categorii/[slug]`
  - redirect din URL-urile vechi `?categorie=` spre ruta canonica `/categorii/[slug]`
  - legaturi interne mutate pe rutele canonice de categorie
- Sitemap extins agresiv:
  - include URL-uri statice importante
  - include toate categoriile indexabile
  - include toate produsele existente
  - include imagini pentru categorii si produse
  - include `priority` si `changeFrequency` ajustate pe intentie
  - include `lastModified` per produs (din `updatedAt`, unde exista)
- Crawl hygiene:
  - `robots.txt` blocheaza `admin` si `api` pentru reducerea crawl waste
- Randare pagini comerciale:
  - eliminat `force-dynamic` pe paginile publice de produs/catalog
  - adaugat `revalidate` + `generateStaticParams` pentru stabilitate SEO/TTFB

## Sitemap Super Detaliat - Acoperire
- URL-uri comerciale:
  - `/produse`
  - `/categorii`
  - `/categorii/{slug}`
  - `/produse/{slug}` pentru toate produsele publice
- Semnale suplimentare:
  - imagini multiple per produs (`featured + gallery`)
  - prioritate ridicata pentru hub-uri comerciale
  - frecventa ridicata pentru pagini care primesc actualizari dese

## Solutii Concrete Urmatoare (SEO Agresiv - Execution Plan)
### Faza 1 (0-14 zile)
- Deschidere Google Search Console + Bing Webmaster (daca nu este deja facut)
- Trimitere sitemap nou si validare indexare pe:
  - `/produse`
  - 10 pagini categorie
  - top 20 produse
- Implementare tracking conversie micro:
  - click WhatsApp
  - click telefon
  - submit formular cerere

### Faza 2 (15-45 zile)
- Creare cluster de pagini comerciale pentru intentie:
  - `mobilier din lemn masiv la comanda [oras]`
  - `mese lemn masiv la comanda`
  - `paturi lemn masiv`
  - `scari interioare din lemn`
  - `riflaje lemn`
- Publicare ghiduri tehnice (2-3/luna) cu interlink catre categorii si produse

### Faza 3 (45-90 zile)
- Link building local + nisa:
  - directoare business locale relevante
  - parteneri arhitecti/designeri
  - portofolii proiecte cu link catre pagina de categorie
- Optimizare CTR:
  - ajustare titluri/meta pe paginile cu impresii mari si CTR mic

## KPI Recomandati
- index coverage: pagini valide in GSC
- impresii non-brand pe categorii si produse
- click-uri organice pe pagini comerciale
- lead-uri organice (form + WhatsApp + telefon)
